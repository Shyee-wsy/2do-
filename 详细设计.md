# 详细设计  
### 一、接口设计  
#### 1. 查  
 get_todo（请求方式：GET）  

| 字段 | 类型 | 描述 | 
| :----: | :----: | :---- | 
| id | Int | 标识每个item |  
| text | string | item的内容 |
| todo_type_id | string | (外键)隶属于的清单名称 | 
| done | Boolean | 是否被标记为完成 |  
| timestamp | DateTime | 时间戳，创建时间 | 
  
 #### 2. 增  
 1. new_list（POST）   
 
| 字段 | 类型 | 描述 |  
| :---: | :---: | :--- |   
| text | string | 清单名称 |    

2. new_todo （POST）  

| 字段 | 类型 | 描述 |  
| :---: | :---: | :--- |
| todo_list_type | string | 清单名称 |  
| text | string | 内容 |  

#### 3. 删  
1. delete_list（DELETE） 

| 字段 | 类型 | 描述 |  
| :---: | :---: | :--- |
| id | int | 列表id |  

2. delete_todo （DELETE） 

| 字段 | 类型 | 描述 |  
| :---: | :---: | :--- |
| id | int | 项目id |  

#### 4. 改（PUT）  

| 字段 | 类型 | 描述 |  
| :---: | :---: | :--- |
| id | int | 项目id |  
| done | Boolean | 标识是否完成 |
